# BRXCE OS — Agentic Operating System | Agentic Workflow Framework

> Your business, run by agentic workflows. Goal-aligned. Self-improving. Human-in-the-loop.

🇰🇷 [한국어 README](README.ko.md)

**BRXCE OS** is an agentic operating system framework that turns AI agents from chat assistants into goal-driven business operators. Built on real-world experience running a 4-person company with AI agents handling strategy alignment, task management, content pipelines, and decision tracking.

## What is BRXCE OS?

BRXCE OS is not a product — it's an **operating framework**. It defines how AI agents should work alongside humans to run a business:

- 🎯 **Goal-aligned execution** — Every action is checked against business objectives
- 🔄 **Self-improving loops** — Weekly agent performance reviews and workflow optimization
- 📋 **Decision logging** — Every important decision tracked with context, alternatives, and outcomes
- 📊 **Performance measurement** — Weekly output tracking, time allocation analysis
- 🔍 **Audit trails** — Full visibility into what agents did and why
- ⚡ **Task auto-prioritization** — Goals cascade into prioritized, actionable tasks

## 7 Operating Principles

1. **No Goal Drift** — Heartbeat monitoring prevents tool/system tinkering without output
2. **Work = Product** — Every workflow with your agent is a case study and content
3. **Stand on Giants' Shoulders** — Always research best practices before starting
4. **Document Everything** — Principles and workflows are recorded and kept current
5. **Daily Backup** — Data safety is non-negotiable
6. **Preempt the Bottleneck** — Humans are the bottleneck; agents provide options and maintain focus
7. **Context Completion** — When requests are vague, verify intent and suggest better alternatives

## 5 Execution Systems

### 1. Decision Log
Track every important decision with full context:
- Why was this decision needed?
- What were the alternatives?
- Why was this option chosen?
- What outcome do we expect?
- When do we review?

### 2. Self-Improvement Loop (Weekly)
- Agent reviews its own performance
- Identifies missed opportunities and mistakes
- Proposes workflow improvements
- Updates operating principles based on lessons learned

### 3. Performance Scorecard (Weekly)
- Output count by business track
- Time allocation: goal-aligned vs. system maintenance vs. other
- Lead/pipeline movement
- Revenue progress toward annual target

### 4. Audit Log
- All automated actions tracked
- External actions (emails, messages, uploads) flagged
- Internal changes (file edits, note creation) recorded
- Anomaly detection and reporting

### 5. Task Auto-Alignment
- Business goals cascade into projects → tasks → subtasks
- Every task checked for goal alignment
- Misaligned tasks flagged for review or deprioritization
- Daily prioritization based on impact and urgency

## Architecture

```
Goals (Annual/Quarterly)
  └── Tracks (Business Lines)
       └── Projects
            └── Tasks (auto-prioritized by goal alignment)

Operating Principles ──→ Agent Behavior
Heartbeat ──→ Goal Alignment Check + Audit + Backup
Decision Log ──→ Institutional Memory
Self-Review ──→ Continuous Improvement
Performance ──→ Accountability
```
